# 입력값 검증 테스트 예시 가이드

입력값 검증 커버리지 체크리스트(2번)의 각 테스트 유형별 구체적인 입력값과 예상 결과를 제공합니다.

---

## 1. 숫자 입력란 검증

### 범위 제한이 있는 경우 (예: 1-999)

| 테스트 유형 | 입력값 | 예상 결과 | 체크리스트 항목 |
|-------------|--------|-----------|----------------|
| 최소값 (경계값 하한) | 1 | 정상 입력, 저장 성공 | 2-4 |
| 최대값 (경계값 상한) | 999 | 정상 입력, 저장 성공 | 2-5 |
| 최소값-1 (경계값 벗어남) | 0 | "1 이상 입력해주세요" 에러 메시지 표시 | 2-6 |
| 최대값+1 (경계값 벗어남) | 1000 | "999 이하 입력해주세요" 에러 메시지 표시 | 2-7 |
| 빈 값 | (공백) | "필수 입력 항목입니다" 또는 필드 테두리 빨간색 표시 | 2-1 |
| 음수 | -10 | "양수만 입력 가능합니다" 에러 메시지 | 2-6 |
| 소수점 | 10.5 | 정수만 허용 시: "정수만 입력 가능합니다" | 2-8 |
| 문자 포함 | 10abc | 입력 차단 또는 "숫자만 입력해주세요" | 2-8 |

### 범위 제한이 없는 경우

| 테스트 유형 | 입력값 | 예상 결과 | 체크리스트 항목 |
|-------------|--------|-----------|----------------|
| 매우 큰 수 | 999999999 | 정상 입력 또는 시스템 최대값 에러 | 2-7 |
| 0 | 0 | 비즈니스 로직에 따라 허용/불허 | 2-4 |

---

## 2. 텍스트 입력란 검증

### 길이 제한이 있는 경우 (예: 최대 50자)

| 테스트 유형 | 입력값 | 예상 결과 | 체크리스트 항목 |
|-------------|--------|-----------|----------------|
| 최소값 | 1자 (예: "a") | 정상 입력 | 2-4 |
| 최대값 | 50자 (예: "a" 50개) | 정상 입력, 저장 성공 | 2-5 |
| 최소값-1 | 빈 값 ("") | "필수 입력 항목입니다" 에러 | 2-1 |
| 최대값+1 | 51자 | 입력 차단 또는 "50자 이하로 입력해주세요" | 2-7 |
| 공백만 입력 | "   " (스페이스 3개) | trim 처리 후 빈 값으로 인식, 에러 | 2-2 |
| 앞뒤 공백 포함 | " abc " | trim 처리되어 "abc"로 저장 | 2-3 |
| 특수문자 | !@#$%^&*() | 허용 여부에 따라 정상 입력 또는 에러 | 2-8 |
| 이모지 | 😀🎉👍 | 허용 여부에 따라 정상 입력 또는 에러 | 2-9 |
| 한글/영문/숫자 혼합 | "홍길동123abc" | 정상 입력 (일반적으로 허용) | 2-10 |
| SQL 인젝션 시도 | ' OR '1'='1 | 입력 차단 또는 이스케이프 처리 후 저장 | 2-8 |
| XSS 시도 | \<script>alert('XSS')\</script> | 입력 차단 또는 이스케이프 처리 후 저장 | 2-8 |

### 이메일 형식 (예: 이메일 입력란)

| 테스트 유형 | 입력값 | 예상 결과 | 체크리스트 항목 |
|-------------|--------|-----------|----------------|
| 유효한 형식 | test@example.com | 정상 입력 | 2-4 |
| @ 누락 | testexample.com | "유효한 이메일을 입력해주세요" | 2-8 |
| 도메인 누락 | test@ | "유효한 이메일을 입력해주세요" | 2-8 |
| 특수문자 포함 | test+tag@example.com | 정상 입력 (+ 허용됨) | 2-10 |

### 전화번호 형식 (예: 010-1234-5678)

| 테스트 유형 | 입력값 | 예상 결과 | 체크리스트 항목 |
|-------------|--------|-----------|----------------|
| 유효한 형식 | 010-1234-5678 | 정상 입력 | 2-4 |
| 하이픈 누락 | 01012345678 | 자동 하이픈 추가 또는 에러 | 2-8 |
| 자릿수 부족 | 010-123-456 | "올바른 전화번호를 입력해주세요" | 2-7 |
| 문자 포함 | 010-abcd-5678 | 입력 차단 또는 에러 | 2-8 |

## 3. 날짜 입력란 검증

### 날짜 범위 제한 (예: 오늘부터 1년 이내)

**기준일**: 2025-01-15 (테스트 실행일)

| 테스트 유형 | 입력값 예시 | 예상 결과 | 체크리스트 항목 |
|-------------|-------------|-----------|----------------|
| 유효한 날짜 (범위 내) | 2025-06-15 | 정상 입력, 저장 성공 | 3-1 |
| 오늘 날짜 | 2025-01-15 | 정상 입력 | 3-6 |
| 과거 날짜 (허용 범위 벗어남) | 2024-01-01 | "오늘 이후 날짜를 선택해주세요" 에러 | 3-5 |
| 미래 날짜 (범위 초과) | 2027-01-01 | "1년 이내 날짜를 선택해주세요" 에러 | 3-4 |
| 잘못된 날짜 형식 | 2025/13/45 | "올바른 날짜를 입력해주세요" 에러 | 3-3 |
| 존재하지 않는 날짜 | 2025-02-30 | "올바른 날짜를 입력해주세요" 에러 | 3-3 |
| 날짜 역순 (시작일 > 종료일) | 시작: 2025-12-31, 종료: 2025-01-01 | "종료일은 시작일 이후여야 합니다" 에러 | 3-2 |
| 빈 값 | (선택 안 함) | 필수 필드 시: "날짜를 선택해주세요" | 2-1 |

### 생년월일 (예: 만 14세 이상)

**기준일**: 2025-01-15

| 테스트 유형 | 입력값 예시 | 예상 결과 | 체크리스트 항목 |
|-------------|-------------|-----------|----------------|
| 만 14세 (경계값) | 2011-01-15 | 정상 입력 | 2-4 |
| 만 14세 미만 (경계값-1일) | 2011-01-16 | "만 14세 이상만 가입 가능합니다" | 2-6 |
| 만 120세 (상한선) | 1905-01-15 | 정상 입력 또는 "올바른 생년월일을 입력해주세요" | 2-5 |
| 미래 날짜 | 2030-01-01 | "미래 날짜는 입력할 수 없습니다" | 3-4 |

## 4. 파일 업로드 검증

### 파일 크기 제한 (예: 최대 10MB)

| 테스트 유형 | 입력값 | 예상 결과 | 체크리스트 항목 |
|-------------|--------|-----------|----------------|
| 최대 크기 이하 | 9.5MB | 정상 업로드 | 2-5 |
| 최대 크기 초과 | 11MB | "파일 크기는 10MB 이하여야 합니다" | 2-7 |
| 빈 파일 | 0KB | "빈 파일은 업로드할 수 없습니다" | 2-1 |

|-------------|--------|-----------|----------------|
| 허용 형식 | image.jpg | 정상 업로드 | 2-4 |
| 허용 형식 | photo.png | 정상 업로드 | 2-4 |
| 비허용 형식 | document.pdf | "jpg, png 파일만 업로드 가능합니다" | 2-8 |

## 5. 비밀번호 입력란 검증

| 테스트 유형 | 입력값 | 예상 결과 | 체크리스트 항목 |
|-------------|--------|-----------|----------------|
| 모든 조건 충족 | Pass1234! | 정상 입력 | 2-4 |

| 최대 길이 (20자) | Pass1234!@#$abcd1234 | 정상 입력 | 2-5 |
| 최소 길이 미만 | Pass12! | "8자 이상 입력해주세요" | 2-6 |

| 숫자 누락 | Password! | "숫자를 포함해주세요" | 2-8 |
| 특수문자 누락 | Pass1234 | "특수문자를 포함해주세요" | 2-8 |
| 영문 누락 | 1234!@#$ | "영문을 포함해주세요" | 2-8 |

## 6. 체크박스/라디오 버튼 검증

### 필수 선택 (예: 약관 동의)

| 테스트 유형 | 입력값 | 예상 결과 | 체크리스트 항목 |
|-------------|--------|-----------|----------------|
| 선택함 | 체크 | 정상 진행 | 2-4 |
| 선택 안 함 | 미체크 | "필수 약관에 동의해주세요" 에러 | 2-1 |

### 다중 선택 (예: 관심 분야)

| 테스트 유형 | 입력값 | 예상 결과 | 체크리스트 항목 |
|-------------|--------|-----------|----------------|
| 0개 선택 | 미선택 | 선택 사항이면 정상, 필수면 에러 | 2-1 |
| 1개 선택 | 1개 체크 | 정상 진행 | 2-4 |
| 전체 선택 | 모두 체크 | 정상 진행 | 2-5 |
| 최대 개수 초과 (제한 있을 시) | 4개 체크 (최대 3개) | "최대 3개까지 선택 가능합니다" | 2-7 |

## 7. 드롭다운/셀렉트박스 검증

| 테스트 유형 | 입력값 | 예상 결과 | 체크리스트 항목 |
|-------------|--------|-----------|----------------|
| 유효한 옵션 선택 | "서울특별시" 선택 | 정상 입력 | 2-4 |
| 미선택 (기본값) | "선택하세요" 상태 | 필수 선택 시 에러 | 2-1 |
| 옵션 개수 (1개) | 1개만 존재 | 자동 선택 또는 선택 필요 | 2-2 |
| 옵션 개수 (100개 이상) | 스크롤 필요 | 정상 동작, 검색 기능 제공 | 2-3 |

## 활용 방법

1. **테스트케이스 작성 시**: 위 표의 "체크리스트 항목" 컬럼을 참조하여 누락된 테스트 확인
2. **Expected Result 작성 시**: "예상 결과" 컬럼의 구체적 메시지를 참고하여 명확하게 작성
3. **Steps 작성 시**: "입력값" 컬럼의 예시를 그대로 또는 변형하여 사용

---

## 주의사항

- 위 예시는 일반적인 케이스이며, 실제 프로젝트의 비즈니스 로직에 따라 달라질 수 있음
- 예상 결과의 에러 메시지는 실제 화면의 메시지와 일치시켜야 함
- 경계값은 프로젝트별로 다르므로 요구사항 문서에서 확인 필수
