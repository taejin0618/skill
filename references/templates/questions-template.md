# 질문 템플릿

요구사항 분석 시 불명확한 항목을 정리하여 이해관계자에게 확인받는다.

---

## 템플릿

```markdown
# [기능명] 이해관계자 질문

## 개요

| 항목 | 내용 |
|------|------|
| **작성 일자** | YYYY-MM-DD |
| **대상 기능** | [기능/화면명] |
| **요구사항 URL** | [Figma/Axure/Jira/Confluence URL 입력] |
| **질문 대상** | 기획자 / 개발자 / 디자이너 |

## 질문 목록

### 🔴 Critical (테스트 진행 불가)
답변 없이는 테스트 케이스 작성이 불가능한 항목

| # | 질문 | 관련 화면/기능 | 대상 | 답변 |
|---|------|---------------|------|------|
| C1 | [질문 내용] | [화면명] | 기획자 | |
| C2 | | | | |

### 🟠 High (주요 시나리오 영향)
핵심 테스트 케이스 작성에 필요한 항목

| # | 질문 | 관련 화면/기능 | 대상 | 답변 |
|---|------|---------------|------|------|
| H1 | [질문 내용] | [화면명] | 기획자 | |
| H2 | | | | |

### 🟡 Medium (엣지케이스 관련)
엣지케이스 및 예외 상황 테스트에 필요한 항목

| # | 질문 | 관련 화면/기능 | 대상 | 답변 |
|---|------|---------------|------|------|
| M1 | [질문 내용] | [화면명] | 개발자 | |
| M2 | | | | |

### 🟢 Low (개선 제안)
테스트 진행에는 영향 없으나 확인이 필요한 항목

| # | 질문 | 관련 화면/기능 | 대상 | 답변 |
|---|------|---------------|------|------|
| L1 | [질문 내용] | [화면명] | 디자이너 | |
| L2 | | | | |

## 가정 사항
답변 전까지 아래 가정을 기반으로 테스트 케이스 작성

| # | 가정 내용 | 관련 질문 | 리스크 |
|---|----------|----------|--------|
| A1 | [가정 내용] | C1 | [가정이 틀렸을 때 영향] |
| A2 | | | |
```

## 질문 우선순위 기준

| 등급 | 기준 | 예시 |
|------|------|------|
| 🔴 **Critical** | 답변 없이 테스트 불가 | 핵심 비즈니스 로직, 필수 입력값, 권한 정책 |
| 🟠 **High** | 주요 시나리오에 영향 | 정상 케이스 분기, 상태 전환 조건, 검증 규칙 |
| 🟡 **Medium** | 엣지케이스에 영향 | 경계값, 예외 처리, 에러 메시지 문구 |
| 🟢 **Low** | 참고용 / 개선 제안 | UI 디자인 의도, 향후 개선 계획 |

## 자주 발생하는 질문 유형

### 1. 비즈니스 로직

| 질문 유형 | 예시 |
|----------|------|
| 권한 정책 | "관리자만 수정 가능한가요, 본인도 수정 가능한가요?" |
| 상태 전환 | "취소된 주문을 다시 진행할 수 있나요?" |
| 계산 로직 | "할인 적용 순서가 쿠폰 먼저인가요, 포인트 먼저인가요?" |
| 중복 처리 | "동일 이메일로 재가입이 가능한가요?" |

### 2. 입력 검증

| 질문 유형 | 예시 |
|----------|------|
| 필수 여부 | "휴대폰 번호는 필수 입력인가요?" |
| 형식 제한 | "비밀번호에 특수문자가 필수인가요?" |
| 길이 제한 | "제목 최대 글자수가 몇 자인가요?" |
| 허용 범위 | "수량에 0이나 음수 입력이 가능한가요?" |

### 3. 화면/UX

| 질문 유형 | 예시 |
|----------|------|
| 기본값 | "검색 기간 기본값이 1개월인가요, 1주일인가요?" |
| 정렬 순서 | "목록 기본 정렬이 최신순인가요, 이름순인가요?" |
| 페이징 | "페이지당 기본 노출 개수가 몇 개인가요?" |
| 에러 메시지 | "중복 이메일 에러 메시지 문구가 정해져 있나요?" |

### 4. 연동/외부 시스템

| 질문 유형 | 예시 |
|----------|------|
| API 응답 | "결제 실패 시 어떤 에러 코드가 반환되나요?" |
| 타임아웃 | "외부 API 타임아웃 설정이 몇 초인가요?" |
| 재시도 | "메일 발송 실패 시 자동 재시도되나요?" |
| 동기/비동기 | "파일 업로드가 동기 처리인가요, 비동기인가요?" |

---

## 질문 작성 팁

1. **구체적으로** - "어떻게 처리되나요?" (X) → "A 상황에서 B가 발생하면 C인가요, D인가요?" (O)
2. **선택지 제시** - 예상 가능한 답변을 선택지로 제시하면 답변 속도 향상
3. **화면 참조** - 관련 화면 캡처 또는 Figma 링크 첨부
4. **영향도 설명** - 답변에 따라 테스트 케이스가 어떻게 달라지는지 명시
