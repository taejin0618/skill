# 리스크 분석 템플릿

테스트 케이스 작성 시 위험 요소를 분석하여 테스트 우선순위를 결정한다.

---

## 템플릿

```markdown
# [기능명] 리스크 분석

## 개요
- **분석 일자**: YYYY-MM-DD
- **분석 대상**: [기능/화면명]
- **분석자**: Claude

## 리스크 매트릭스

| # | 위험 항목 | 발생 가능성 | 영향도 | 리스크 등급 | 완화 전략 |
|---|----------|------------|--------|------------|----------|
| 1 | [구체적 위험 상황] | 높음/중간/낮음 | 높음/중간/낮음 | Critical/High/Medium/Low | [구체적 대응 방안] |
| 2 | | | | | |

## 주요 리스크 상세

### [리스크 #1]: [위험 항목명]
- **설명**: [상세 설명]
- **발생 조건**: [어떤 상황에서 발생하는지]
- **영향 범위**: [영향 받는 사용자/기능]
- **완화 전략**: 
  1. [전략 1]
  2. [전략 2]
- **담당자/일정**: [필요시 기재]

## 테스트 우선순위 권고
1. Critical 등급 리스크 관련 테스트 케이스 우선 실행
2. High 등급 리스크는 회귀 테스트에 필수 포함
3. Medium/Low는 시간 여유 시 실행
```

---

## 등급 산정 기준

### 발생 가능성 기준

| 등급 | 기준 | 예시 |
|------|------|------|
| **높음** | 80% 이상 발생 예상 | 유사 기능에서 빈번히 발생, 복잡한 로직 |
| **중간** | 30~80% 발생 예상 | 특정 조건에서 발생, 외부 연동 의존 |
| **낮음** | 30% 미만 발생 예상 | 극히 드문 상황, 단순 로직 |

### 영향도 기준

| 등급 | 기준 | 예시 |
|------|------|------|
| **높음** | 서비스 중단, 데이터 손실 | 결제 오류, 개인정보 유출, 매출 영향, 법적 이슈 |
| **중간** | 주요 기능 장애 | 핵심 기능 오작동, 사용자 불편, 대안 존재 |
| **낮음** | 경미한 오류 | UI 깨짐, 오타, 사용성 저하 |

### 리스크 등급 매트릭스

| 발생 가능성 \ 영향도 | 높음 | 중간 | 낮음 |
|---------------------|------|------|------|
| **높음** | 🔴 Critical | 🟠 High | 🟡 Medium |
| **중간** | 🟠 High | 🟡 Medium | 🟢 Low |
| **낮음** | 🟡 Medium | 🟢 Low | 🟢 Low |

## 자주 발생하는 리스크 유형

### 1. 데이터 관련
- 데이터 손실/유실
- 중복 데이터 생성
- 데이터 불일치 (화면 vs DB)
- 대용량 데이터 처리 실패

### 2. 보안 관련
- 인증/인가 우회
- 개인정보 노출
- SQL Injection / XSS
- 세션 탈취

### 3. 성능 관련
- 응답 지연 (3초 이상)
- 동시 요청 처리 실패
- 메모리 누수
- 무한 루프

### 4. 연동 관련
- 외부 API 장애
- 결제 모듈 오류
- 메일/SMS 발송 실패
- 파일 업로드/다운로드 실패

### 5. UX 관련
- 에러 메시지 미표시
- 로딩 상태 미표시
- 뒤로가기 시 데이터 손실
- 모바일 호환성 문제
