# Happy Case 분류 기준 상세 가이드

AI가 테스트케이스를 분류할 때 참조하는 상세 기준이다.
판단 순서: Expected Result → Title → Steps (이 순서로 체크)

---

## 1. Happy Case 포함 기준

아래 3가지 필드를 종합하여 판단한다. 하나의 필드만으로 판단하지 않는다.

### 1-1. Title 기준 (포함 키워드)

| 카테고리 | 키워드 | 예시 Title |
|----------|--------|-----------|
| 긍정 수식어 | 유효한, 정상적인, 올바른 | "유효한 이메일로 가입" |
| 성공 상태 | 성공, 완료, 정상, 확인 | "로그인 성공 후 대시보드 이동" |
| 일반 동작 | 조회, 검색, 등록, 저장, 수정, 삭제 | "회원 정보 정상 저장" |
| UI 동작 | 클릭, 선택, 입력, 이동, 표시, 열기, 닫기 | "검색 버튼 클릭 정상 동작" |
| 파일 동작 | 다운로드, 업로드, 내보내기, 가져오기 | "엑셀 다운로드 정상 동작" |

### 1-2. Steps 기준 (Happy Case 특성)

- 구체적인 유효 입력값 사용 (예: `'test@example.com'`, `'홍길동'`, `'2025-01-15'`)
- 정상적인 순서로 진행 (접속 → 입력 → 버튼 클릭 → 결과 확인)
- 표준 사용자 행동 패턴 준수
- 1건 이상의 유효 데이터 사용

### 1-3. Expected Result 기준 (반드시 성공 결과)

| 성공 패턴 | 예시 |
|-----------|------|
| 성공/완료 메시지 | `'저장되었습니다'`, `'등록 완료'`, `'처리되었습니다'`, `'삭제되었습니다'` |
| 페이지 이동 | `대시보드로 이동`, `목록 페이지로 이동`, `URL 변경됨` |
| 데이터 표시 | `목록에 추가됨`, `화면에 표시됨`, `검색 결과 N건 표시` |
| 로딩 완료 | `로딩 인디케이터 표시 후 사라짐` |
| 정상 상태 | `버튼 활성화됨`, `체크박스 선택됨`, `탭 전환됨` |

---

## 2. Error Case 제외 기준

다음 중 하나라도 해당되면 EXCLUDE 처리한다.

### 2-1. Title 제외 키워드

| 키워드 유형 | 키워드 목록 |
|-------------|------------|
| 오류/실패 명시 | 오류, 에러, 실패, error, fail |
| 부정 입력 수식어 | 잘못된, 유효하지 않은, 올바르지 않은, 틀린 |
| 빈 값 | 빈 값, 빈칸, 미입력, 입력하지 않고, 입력 없이, 미선택 |
| 거부/차단 | 접근 거부, 권한 없음, 권한없음, 차단, 금지 |
| 중복 | 중복, 이미 사용 중, 이미 존재, 재시도 |

### 2-2. Expected Result 제외 패턴

| 패턴 | 예시 |
|------|------|
| 오류 메시지 | `'올바르지 않습니다'`, `'오류가 발생했습니다'`, `'다시 시도해주세요'` |
| 필수 입력 오류 | `'필수 입력'`, `'입력해주세요'`, `'필수 항목'` |
| 시각적 오류 표시 | `빨간색 테두리`, `붉은 테두리`, `경고 아이콘`, `오류 아이콘` |
| 접근 차단 | `403`, `404`, `500`, `접근 불가`, `페이지를 찾을 수 없습니다` |
| 재입력 요구 | `재입력`, `다시 입력`, `수정 후 재시도` |

### 2-3. Steps 제외 패턴

| 패턴 | 예시 Steps |
|------|-----------|
| 빈 값 입력 | "이메일 입력란을 **비워두고**", "**아무것도 입력하지 않고**" |
| 잘못된 형식 | "**잘못된 이메일 형식**으로 입력", "숫자 아닌 값으로" |
| 범위 초과 | "**최대 글자수 초과** 입력", "**100자 이상** 입력" |
| 비정상 접근 | "**권한 없는 상태**로 메뉴 접근", "**로그아웃 상태**에서 URL 직접 접속" |
| 특수 입력 | "`<script>` 입력", "`' OR '1'='1` 입력", "이모지 입력" |

## 3. Edge Case 제외 기준

### 3-1. Edge Case 정의

경계값, 예외 상황, 비정상 상태에서의 동작 테스트 케이스

### 3-2. Edge Case 제외 키워드/패턴

| 카테고리 | 키워드/패턴 |
|----------|------------|
| 경계값 | 최댓값, 최솟값, 경계값, 경계, 딱 N자, N자 초과, N자 미만 |
| 특수 데이터 | 특수문자, 이모지, 한자, 아랍어, 공백만, 줄바꿈만 |
| 비정상 상태 | 데이터 없음, 0건, 빈 목록, 연결 끊김 |
| 동시성 | 동시에, 중복 요청, 연속 클릭, 이중 제출 |
| 브라우저 동작 | 뒤로가기, 새로고침 후, 브라우저 닫기, 탭 전환 |
| 세션/타임아웃 | 세션 만료, 타임아웃, 토큰 만료 |

---

## 4. 판단 우선순위 (체크 순서)

```
Step 1: Expected Result 확인
  └─ '오류', '에러', '실패', '올바르지 않습니다', 빨간 테두리 등 → EXCLUDE

Step 2: Title 확인
  └─ '잘못된', '실패', '오류', '빈 값', '중복', '경계' 등 → EXCLUDE

Step 3: Steps 확인
  └─ '비워두고', '잘못된 값', '특수문자', '동시에' 등 → EXCLUDE

Step 4: 위 3단계 해당 없음 → HAPPY
```

---

## 5. Section 행 처리 규칙

```
규칙 1: Title이 비어있는 행 = Section 헤더 행
규칙 2: Section 행 자체는 HAPPY/EXCLUDE 분류 대상 아님
규칙 3: 해당 Section 내에 Happy Case가 1개 이상 있으면 Section 행 포함
규칙 4: 해당 Section 내 모든 케이스가 EXCLUDE이면 Section 행도 제외
규칙 5: Section Hierarchy 경로상의 모든 상위 Section 행도 포함
```

## 6. 애매한 케이스 처리 기준

| 유형 | 판단 | 근거 |
|------|------|------|
| 삭제 확인 다이얼로그 후 삭제 성공 | **HAPPY** | 정상 UX 플로우, 성공 메시지로 종료 |
| 취소 버튼 클릭 → 이전 화면 이동 | **HAPPY** | 정상 사용자 행동 |
| 관리자 권한으로 설정 메뉴 접근 | **HAPPY** | 유효 권한 사용, 성공 접근 |
| 전체 선택 체크박스로 일괄 선택 | **HAPPY** | 정상 기능 동작 |
| 비밀번호 변경 성공 | **HAPPY** | 유효 값 사용, 성공 메시지 |
| 0건 검색 결과 표시 | **EXCLUDE** | 경계 상황 (빈 상태) |
| 빈 목록 "데이터 없음" 메시지 | **EXCLUDE** | 예외 상태 |
| 경계값(최대 N자) 정상 저장 | **EXCLUDE** | Title/Steps에 '최대', '경계' 키워드 |
| 세션 만료 후 재로그인 | **EXCLUDE** | 비정상 상태 진입 |

---

## 7. 행별 분류 체크리스트 (빠른 판단)

```
□ Expected Result:
  - '오류/에러/실패' 패턴? → EXCLUDE
  - '성공/완료/이동/표시' 패턴? → HAPPY 방향

□ Title:
  - '잘못된/오류/실패/빈 값/중복/경계' 키워드? → EXCLUDE
  - 위 키워드 없음? → 다음 단계

□ Steps:
  - '비워두고/잘못된 값/특수문자/동시에' 패턴? → EXCLUDE
  - 유효한 구체값 사용? → HAPPY 방향

□ 종합:
  - 3가지 모두 이상 없음 → HAPPY
  - 하나라도 Exclude 시그널 → EXCLUDE
```
