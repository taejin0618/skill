# Nielsen 10가지 사용성 원칙 (UX 테스트)

UI/UX 테스트 케이스 작성 시 아래 원칙별 검증 항목 확인.

## 원칙별 검증 체크리스트

| # | 원칙 | 핵심 검증 항목 |
|---|------|---------------|
| 1 | **시스템 상태 가시성** | □ 로딩 인디케이터 (2-3초 내) □ 진행 상태 바 □ 성공/실패 즉시 피드백 □ 현재 위치 표시 |
| 2 | **현실 세계 일치** | □ 친숙한 용어 사용 □ 아이콘-기능 일치 □ 자연스러운 순서/흐름 |
| 3 | **사용자 제어와 자유** | □ 되돌리기/취소 기능 □ 뒤로가기 □ 모달 닫기 (ESC 지원) □ 작업 중단 후 재개 |
| 4 | **일관성과 표준** | □ UI 컴포넌트 스타일 일관성 □ 버튼 위치 일관성 □ 용어 일관성 □ 플랫폼 표준 준수 |
| 5 | **오류 예방** | □ 중요 작업 확인 다이얼로그 □ 실시간 입력 검증 □ 비활성화 상태 표시 □ 기본값 제공 |
| 6 | **기억보다 인식** | □ 시각적 힌트 (아이콘+텍스트) □ 자동완성/제안 □ 최근 사용 항목 표시 |
| 7 | **유연성과 효율성** | □ 단축키 지원 □ 빠른 실행 메뉴 □ 커스터마이징 옵션 □ 배치 작업 |
| 8 | **미니멀 디자인** | □ 불필요한 요소 제거 □ 핵심 콘텐츠 강조 □ 여백 활용 □ 시각적 계층 구조 |
| 9 | **오류 복구** | □ 명확한 오류 메시지 □ 해결 방법 제시 □ 오류 위치 표시 □ 복구 옵션 제공 |
| 10 | **도움말** | □ 도움말 접근성 □ 검색 가능한 FAQ □ 컨텍스트 툴팁 □ 온보딩 튜토리얼 |
