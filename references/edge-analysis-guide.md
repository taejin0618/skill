# 엣지케이스 분석 가이드

PASS 2b (findChunkEdges)에서 참조하는 분석 관점 체크리스트.

## 1. 경계값 (Boundary Values)

기획서에 수량, 금액, 기간 등 수치가 등장하면 다음을 확인:

| 체크 항목 | 질문 |
|-----------|------|
| 최솟값 | 0 또는 1일 때 동작이 정의되어 있는가? |
| 최댓값 | 상한이 명시되어 있는가? 초과 시 동작은? |
| 음수 | 음수 입력 가능성이 고려되어 있는가? |
| 소수점 | 소수점 처리 규칙이 있는가? (반올림/절사) |
| 시간 경계 | 자정, 월말, 연말 등 경계 시점 처리는? |
| 단위 불일치 | 같은 값에 다른 단위가 혼용되고 있지 않은가? |

## 2. 동시성 (Concurrency)

| 체크 항목 | 질문 |
|-----------|------|
| 동시 접근 | 같은 리소스에 복수 사용자가 동시에 접근하면? |
| 동시 수정 | 같은 데이터를 동시에 수정하면 어느 것이 우선? |
| 순서 의존 | 특정 순서로만 동작해야 하는데 순서가 보장되는가? |
| 중복 요청 | 같은 요청을 빠르게 2번 보내면? (더블클릭 등) |

## 3. 예외 흐름 (Exception Flow)

| 체크 항목 | 질문 |
|-----------|------|
| 네트워크 에러 | 통신 실패 시 재시도/에러 표시 정의가 있는가? |
| 타임아웃 | 응답 지연 시 동작이 정의되어 있는가? |
| 취소/중단 | 진행 중 취소 시 롤백 범위가 명확한가? |
| 부분 실패 | 복수 항목 처리 중 일부만 실패하면? |
| 세션 만료 | 작업 중 세션이 만료되면 데이터 보존은? |

## 4. 권한 교차 (Permission Intersection)

| 체크 항목 | 질문 |
|-----------|------|
| 복수 역할 | 한 사용자가 여러 역할을 가지면? |
| 권한 전환 | 역할이 변경될 때 진행 중 작업은? |
| 위임 | 다른 사용자에게 권한 위임이 가능한가? |
| 관리자 예외 | 관리자가 일반 규칙을 우회할 수 있는가? |

## 5. 데이터 없음 (Empty/Null States)

| 체크 항목 | 질문 |
|-----------|------|
| 0건 목록 | 리스트가 비어있을 때 UI와 동작은? |
| 미입력 | 선택/필수 필드가 비어있을 때 처리는? |
| 삭제된 참조 | 참조하던 데이터가 삭제되면? |
| 초기 상태 | 최초 사용 시(데이터 없는 상태) 화면은? |

## 6. 외부 의존성 (External Dependencies)

| 체크 항목 | 질문 |
|-----------|------|
| 외부 API | 외부 서비스 장애 시 대응 방안이 있는가? |
| 결제 연동 | 결제 성공/실패/부분환불 시나리오가 모두 있는가? |
| 알림 채널 | 푸시/SMS/이메일 발송 실패 시 대안이 있는가? |
| 데이터 동기화 | 외부 시스템과 데이터 불일치 시 처리는? |

## 모호성 점수 기준

| 점수 | 판정 | 기준 |
|------|------|------|
| 0~3 | 양호 | 주요 시나리오와 예외 흐름이 명확히 기술됨 |
| 4~6 | 명확화 권장 | 일부 조건 또는 예외 흐름이 암묵적이거나 불완전 |
| 7~10 | 질문 필요 | 핵심 동작이 해석에 따라 달라질 수 있는 수준 |

점수 산정 시 고려 요소:
- "~할 수 있다" 등 선택적 표현의 빈도
- 조건 분기에서 else 케이스 누락
- "적절한", "필요시", "등" 같은 모호한 수식어 사용
- 구체적 수치/기준 없이 정성적 표현만 사용
- 여러 해석이 가능한 문장 구조
