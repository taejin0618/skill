# 분류 예시 가이드

AI가 실제로 분류할 때 참고하는 구체적인 예시 모음이다.
판단이 어려운 케이스를 처리할 때 이 파일을 참조한다.

---

## Happy Case 예시 (HAPPY - 포함)

### 예시 1: 로그인 성공

```
Title: 유효한 계정으로 로그인 성공
Steps:
1. 로그인 페이지 접속 (URL: /login)
2. 이메일 입력란에 'test@example.com' 입력
3. 비밀번호 입력란에 'Pass1234!' 입력
4. [로그인] 버튼 클릭
Expected Result:
1. 대시보드 페이지로 이동 (URL: /dashboard)
2. 우측 상단에 'test@example.com' 표시
3. '로그인 성공' 토스트 메시지 표시

분류: HAPPY
근거: 유효한 값 사용, Expected Result에 성공 메시지 + 페이지 이동
```

### 예시 2: 회원 등록 저장

```
Title: 신규 회원 정보 입력 후 저장
Steps:
1. 회원 등록 화면 접속
2. 이름 입력란에 '홍길동' 입력
3. 이메일 입력란에 'hong@test.com' 입력
4. [저장] 버튼 클릭
Expected Result:
1. '저장되었습니다' 성공 메시지 표시
2. 회원 목록으로 이동
3. '홍길동' 회원이 목록에 표시

분류: HAPPY
근거: 구체적 유효값 사용, '저장되었습니다' 성공 메시지
```

### 예시 3: 삭제 확인 다이얼로그 후 삭제 완료 (주의: HAPPY)

```
Title: 회원 삭제 확인 다이얼로그 표시 후 삭제 완료
Steps:
1. 삭제할 회원 선택
2. [삭제] 버튼 클릭
3. 확인 다이얼로그에서 [확인] 클릭
Expected Result:
1. '정말 삭제하시겠습니까?' 다이얼로그 표시
2. [확인] 클릭 시 삭제 처리
3. '삭제되었습니다' 성공 메시지 표시

분류: HAPPY
근거: 정상 UX 플로우 (확인 후 삭제), Expected Result가 성공 메시지로 종료
```

### 예시 4: 취소 버튼 정상 이동

```
Title: 취소 버튼 클릭 시 목록으로 이동
Steps:
1. 회원 등록 화면 접속
2. [취소] 버튼 클릭
Expected Result:
1. 회원 목록 화면으로 이동
2. 입력 데이터 미저장

분류: HAPPY
근거: 취소는 정상 사용자 행동, Expected Result가 정상 이동
```

### 예시 5: 엑셀 다운로드

```
Title: 엑셀 다운로드 정상 동작
Steps:
1. 회원 목록 화면 접속
2. [엑셀 다운로드] 버튼 클릭
Expected Result:
1. 다운로드 시작
2. 파일명 '회원목록_YYYYMMDD.xlsx' 형식으로 저장

분류: HAPPY
근거: 정상 기능 사용, 성공적 다운로드 결과
```

## Error Case 예시 (EXCLUDE - 제외)

### 예시 6: 잘못된 비밀번호 로그인 실패

```
Title: 잘못된 비밀번호 입력 시 로그인 실패
Steps:
1. 이메일 입력란에 'test@example.com' 입력
2. 비밀번호 입력란에 'wrongpass' 입력
3. [로그인] 버튼 클릭
Expected Result:
1. '아이디 또는 비밀번호가 올바르지 않습니다' 에러 메시지
2. 비밀번호 입력란 빨간색 테두리 표시

분류: EXCLUDE (Error Case)
근거: Title에 '잘못된', '실패' 키워드, Expected Result에 오류 메시지 + 빨간 테두리
```

### 예시 7: 필수 입력 누락 오류

```
Title: 이메일 미입력 시 오류 메시지 표시
Steps:
1. 이메일 입력란을 비워두고
2. [저장] 버튼 클릭
Expected Result:
1. '이메일을 입력해주세요' 필수 입력 오류 메시지
2. 이메일 입력란 빨간 테두리 표시

분류: EXCLUDE (Error Case)
근거: Steps에 '비워두고'(빈 값 패턴), Expected Result에 오류 메시지 + 빨간 테두리
```

### 예시 8: 중복 이메일 오류

```
Title: 중복 이메일 등록 시 오류 처리
Steps:
1. 이미 등록된 'exist@test.com' 이메일로 회원 등록 시도
2. [저장] 버튼 클릭
Expected Result:
1. '이미 사용중인 이메일입니다' 오류 메시지
2. 저장 실패

분류: EXCLUDE (Error Case)
근거: Title에 '중복', Expected Result에 오류 메시지, '실패' 키워드
```

## Edge Case 예시 (EXCLUDE - 제외)

### 예시 9: 경계값 테스트 (성공 결과여도 제외)

```
Title: 이름 입력 최대 50자 경계값 테스트
Steps:
1. 이름 입력란에 정확히 50자 입력
2. [저장] 버튼 클릭
Expected Result:
1. 50자 이름 정상 저장
2. '저장되었습니다' 메시지 표시

분류: EXCLUDE (Edge Case)
근거: Title에 '최대', '경계값' 키워드 → 성공 결과여도 Edge Case로 제외
```

### 예시 10: 0건 검색 결과

```
Title: 검색 결과 0건 시 빈 목록 메시지 표시
Steps:
1. 존재하지 않는 '홍길동123' 이름으로 검색
Expected Result:
1. '검색 결과가 없습니다' 메시지 표시
2. 목록 영역 비어있음

분류: EXCLUDE (Edge Case)
근거: '0건' 경계 상황, Steps에 '존재하지 않는' 패턴, Expected Result가 빈 상태 메시지
```

### 예시 11: 동시 수정 충돌

```
Title: 동일 데이터 동시 수정 시 충돌 처리
Steps:
1. 두 브라우저에서 동시에 같은 회원 데이터 수정
Expected Result:
1. 나중에 저장한 사용자에게 충돌 안내 메시지

분류: EXCLUDE (Edge Case)
근거: '동시' 동시성 테스트, 비정상 상황
```

## 판단 어려운 케이스 처리 예시

### 예시 12: 관리자 권한 접근 → HAPPY

```
Title: 관리자 권한으로 설정 메뉴 접근
Steps:
1. 관리자 계정으로 로그인
2. 설정 메뉴 클릭
Expected Result:
1. 설정 화면 정상 표시

분류: HAPPY
근거: 유효한 권한(관리자) 사용, 오류 없이 성공 접근
```

### 예시 13: 전체 선택 체크박스 → HAPPY

```
Title: 전체 선택 체크박스로 일괄 선택
Steps:
1. 목록 페이지 접속 (데이터 10건)
2. 전체 선택 체크박스 클릭
Expected Result:
1. 모든 10건 항목 체크됨
2. '10개 선택됨' 표시

분류: HAPPY
근거: 정상 일괄 선택 동작, 성공적 결과
```

### 예시 14: 비밀번호 변경 성공 → HAPPY

```
Title: 유효한 새 비밀번호로 변경 성공
Steps:
1. 현재 비밀번호 'Pass1234!' 입력
2. 새 비밀번호 'NewPass5678!' 입력
3. 새 비밀번호 확인란에 'NewPass5678!' 입력
4. [변경] 버튼 클릭
Expected Result:
1. '비밀번호가 변경되었습니다' 성공 메시지

분류: HAPPY
근거: 유효한 값 모두 사용, '유효한' 수식어, 성공 메시지
```

### 예시 15: 비밀번호 불일치 → EXCLUDE

```
Title: 새 비밀번호 확인란 불일치 시 오류
Steps:
1. 새 비밀번호 'NewPass5678!' 입력
2. 확인란에 다른 값 'DiffPass9999!' 입력
3. [변경] 버튼 클릭
Expected Result:
1. '비밀번호가 일치하지 않습니다' 오류 메시지

분류: EXCLUDE (Error Case)
근거: Title에 '불일치', Expected Result에 오류 메시지
```

## 빠른 판단 요약

```
Expected Result 먼저 체크:
  '오류', '에러', '실패', '올바르지 않습니다', '빨간 테두리', '거부', '경고 메시지' → EXCLUDE

Title 체크:
  '잘못된', '실패', '오류', '빈 값', '중복', '경계값', '최대값', '초과', '만료', '누락' → EXCLUDE

Steps 체크:
  '비워두고', '잘못된 값', '특수문자', '동시에', '권한 없는 상태', '존재하지 않는', '만료된', '삭제된' → EXCLUDE

위 3가지 없으면 → HAPPY
(삭제, 취소, 관리자 접근 등 정상 플로우는 HAPPY)
```
